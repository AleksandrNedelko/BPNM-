## Что это

Файл `seo_sup_process.bpmn` — черновой бизнес‑процесс в формате **BPMN 2.0 XML** для взаимодействия:

- **Постановщик**: SEO-менеджер / SEO-специалист (отдел SEO)
- **Принимающая сторона**: SUP/SEO-менеджер (отдел SUP)

В схеме заложены типовые контуры:

- постановка задачи → проверка полноты → уточнения (цикл)
- выполнение → отправка результата → принятие / доработка (цикл) → подтверждение закрытия

## Как открыть

- **Camunda Modeler**: `File → Open File…` и выбрать `seo_sup_process.bpmn`
- **bpmn.io**: открыть онлайн‑редактор и импортировать файл

## Что править под ваш регламент

Обычно потребуется уточнить:

- список обязательных полей задачи (KPI/URL/доступы/приоритет)
- SLA на уточнения/выполнение
- кто финально принимает результат (роль/подразделение)
